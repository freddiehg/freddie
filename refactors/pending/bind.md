# bind: accumulation

This is the complete, explicit design of accumulation in the binding layer: every type, every trait, the free fn, the generated `EventHandler` impls, the entry point, and a fully worked example with the resulting sets. Dispatch (running a matched handler) is not covered here. The outer loop (the merged event stream, diffing the registered set against the OS, the registrar) is in `freddie-keys-plan.md`.

bind is a crate within freddie: `bind` (the traits, the free fn, the entry point) plus `bind_macro` (the `#[derive(Bind)]`). It builds on laserbeam: the edges accumulation recurses are laserbeam's `#[resolve_into]` fields and enum variants.

No clobbering: a trigger is bound at most once on any active path. A child cannot rebind a trigger an ancestor already bound; that is an error, not an override.

## What the app defines

The app defines its source types, one unified `Trigger` enum over them, and a marker that names that enum.

```rust
use std::collections::HashSet;
use std::hash::Hash;

// one struct per source. They are used as set keys, so they derive Hash and Eq.
#[derive(Clone, PartialEq, Eq, Hash)]
struct Keyboard(String);
impl Keyboard { fn new(k: &str) -> Self { Keyboard(k.to_owned()) } }

#[derive(Clone, PartialEq, Eq, Hash)]
struct Foreground(String);
impl Foreground { fn new(app: &str) -> Self { Foreground(app.to_owned()) } }

// the unified trigger enum, one variant per source. derive_more::From gives
// `Keyboard -> Trigger` and `Foreground -> Trigger`, so `.into()` lifts either.
#[derive(Clone, PartialEq, Eq, Hash, derive_more::From)]
enum Trigger {
    Keyboard(Keyboard),
    Foreground(Foreground),
}

// the marker. It names the Trigger enum for the whole app.
struct MercuryStruct;
impl Bindings for MercuryStruct {
    type Trigger = Trigger;
}
```

`Event` and `Output` (the handler's argument and return) belong to dispatch and are not part of `Bindings` here; accumulation never touches handlers, only triggers.

## What bind provides

```rust
// the marker trait the app implements on MercuryStruct.
trait Bindings {
    type Trigger: Clone + Eq + Hash;
}

// every bindable node implements this, generated by #[derive(Bind)].
trait EventHandler<M: Bindings> {
    fn accumulate(&self, out: &mut HashSet<M::Trigger>) -> Result<(), BindError>;
}

// the only accumulation error.
#[derive(Debug)]
enum BindError {
    DuplicateTrigger,   // a trigger bound twice on the active path
}

// the duplicate check. insert returns false when the key is already present
// (std HashSet has no fallible insert), so this turns that into an error.
fn insert_or_error<T: Clone + Eq + Hash>(out: &mut HashSet<T>, t: T) -> Result<(), BindError> {
    if out.insert(t) { Ok(()) } else { Err(BindError::DuplicateTrigger) }
}

// the entry point: allocate the set, run the root's accumulate, return the set.
fn accumulate<M, N>(root: &N) -> Result<HashSet<M::Trigger>, BindError>
where
    M: Bindings,
    N: EventHandler<M>,
{
    let mut out = HashSet::new();
    root.accumulate(&mut out)?;
    Ok(out)
}
```

The app calls `bind::accumulate::<MercuryStruct, _>(&root)` to get the `HashSet<Trigger>` for the current state.

## The attributes

```rust
#[derive(Laserbeam, Bind)]
#[laserbeam(path = NavPath, resolved = Resolved)]   // laserbeam's own attributes
#[binds(MercuryStruct)]                              // names the marker M
#[bind(Keyboard::new("g"), open_chrome)]            // a binding: trigger, handler
struct Nav {}
```

- `#[binds(MercuryStruct)]`: names the marker. The derive emits `impl EventHandler<MercuryStruct> for Nav`. Because `M` is the concrete `MercuryStruct`, `M::Trigger` is the concrete `Trigger`, so `out` is `&mut HashSet<Trigger>` and `.into()` lifts each trigger into `Trigger`.
- `#[bind(trigger, handler)]`, repeatable. `trigger` is any expression lifting into `Trigger` via `Into`; accumulation uses only the trigger. `handler` is recorded for dispatch and is otherwise unused here.

## What is bindable

Accumulation recurses laserbeam's descent edges, nothing else:

- For a struct, the bindable children are its `#[resolve_into]` fields. Plain fields (`String`, counters, buffers) are state; they are never recursed into and never implement `EventHandler`.
- For an enum, the bindable child is the active variant's inner type. Every variant's inner type implements `EventHandler<M>` (it derives `Bind`).

So `EventHandler<M>` is implemented only on the node types that derive `Bind`, and accumulation follows only `#[resolve_into]` fields and variant inners.

## What the derive generates

For each node the derive emits one `accumulate`: it inserts the node's own binds with `insert_or_error(..)?`, then recurses into its bindable children, returning their result. A node inserts its own binds before recursing, so a child rebinding an ancestor's trigger hits the already-present key and errors.

## Worked example

The state tree. A root struct holding the active layer, an enum of two layers, and the two leaf layers, each with binds at every level:

```rust
#[derive(Laserbeam, Bind)]
#[laserbeam_root(resolved = Resolved)]
#[binds(MercuryStruct)]
#[bind(Keyboard::new("esc"), to_nav)]        // global: esc returns to nav from anywhere
struct Mercury {
    #[resolve_into] layer: Layer,            // the one bindable child
}

#[derive(Laserbeam, Bind)]
#[laserbeam(path = LayerPath, resolved = Resolved)]
#[binds(MercuryStruct)]
#[bind(Keyboard::new("f1"), show_help)]      // enum-level: f1 in any layer
enum Layer {
    Nav(Nav),
    Typing(Typing),
}

#[derive(Laserbeam, Bind)]
#[laserbeam(path = NavPath, resolved = Resolved)]
#[binds(MercuryStruct)]
#[bind(Keyboard::new("g"), open_chrome)]
#[bind(Keyboard::new("j"), focus_down)]
#[bind(Foreground::new("Slack"), on_slack)]
struct Nav {}

#[derive(Laserbeam, Bind)]
#[laserbeam(path = TypingPath, resolved = Resolved)]
#[binds(MercuryStruct)]
#[bind(Keyboard::new("backspace"), delete_char)]
struct Typing {}
```

The four `EventHandler<MercuryStruct>` impls the derive generates, in full:

```rust
impl EventHandler<MercuryStruct> for Mercury {
    fn accumulate(&self, out: &mut HashSet<Trigger>) -> Result<(), BindError> {
        insert_or_error(out, Keyboard::new("esc").into())?;   // #[bind(.., to_nav)]
        self.layer.accumulate(out)                            // recurse #[resolve_into] layer
    }
}

impl EventHandler<MercuryStruct> for Layer {
    fn accumulate(&self, out: &mut HashSet<Trigger>) -> Result<(), BindError> {
        insert_or_error(out, Keyboard::new("f1").into())?;    // #[bind(.., show_help)]
        match self {                                          // recurse the active variant
            Layer::Nav(inner)    => inner.accumulate(out),
            Layer::Typing(inner) => inner.accumulate(out),
        }
    }
}

impl EventHandler<MercuryStruct> for Nav {
    fn accumulate(&self, out: &mut HashSet<Trigger>) -> Result<(), BindError> {
        insert_or_error(out, Keyboard::new("g").into())?;        // open_chrome
        insert_or_error(out, Keyboard::new("j").into())?;        // focus_down
        insert_or_error(out, Foreground::new("Slack").into())?;  // on_slack
        Ok(())                                                   // leaf: no #[resolve_into]
    }
}

impl EventHandler<MercuryStruct> for Typing {
    fn accumulate(&self, out: &mut HashSet<Trigger>) -> Result<(), BindError> {
        insert_or_error(out, Keyboard::new("backspace").into())?;  // delete_char
        Ok(())
    }
}
```

### State A: the nav layer

`root = Mercury { layer: Layer::Nav(Nav {}) }`, calling `bind::accumulate::<MercuryStruct, _>(&root)`:

1. `out = {}`.
2. `Mercury::accumulate`: insert `Keyboard("esc")` -> `{esc}`. Recurse `self.layer.accumulate(out)`.
3. `Layer::accumulate`: insert `Keyboard("f1")` -> `{esc, f1}`. Active variant is `Nav`, recurse `inner.accumulate(out)`.
4. `Nav::accumulate`: insert `Keyboard("g")` -> `{esc, f1, g}`; insert `Keyboard("j")` -> `{esc, f1, g, j}`; insert `Foreground("Slack")` -> `{esc, f1, g, j, Slack}`. Return `Ok(())`.

Result:

```
Ok({ Keyboard("esc"), Keyboard("f1"), Keyboard("g"), Keyboard("j"), Foreground("Slack") })
```

### State B: the typing layer

`root = Mercury { layer: Layer::Typing(Typing {}) }`:

1. `out = {}`.
2. `Mercury::accumulate`: insert `esc` -> `{esc}`. Recurse.
3. `Layer::accumulate`: insert `f1` -> `{esc, f1}`. Active variant is `Typing`, recurse.
4. `Typing::accumulate`: insert `backspace` -> `{esc, f1, backspace}`. Return `Ok(())`.

Result:

```
Ok({ Keyboard("esc"), Keyboard("f1"), Keyboard("backspace") })
```

### The collision case

Suppose `Typing` instead bound `esc`:

```rust
#[bind(Keyboard::new("esc"), stop_typing)]
struct Typing {}
```

Accumulating state B now:

1. `Mercury::accumulate`: insert `esc` -> `{esc}`. Recurse.
2. `Layer::accumulate`: insert `f1` -> `{esc, f1}`. Recurse into `Typing`.
3. `Typing::accumulate`: `out.insert(Keyboard("esc").into())` returns `false`, because the root already inserted `esc`. `insert_or_error` returns `Err(BindError::DuplicateTrigger)`, which `?` propagates out of `Typing::accumulate`, `Layer::accumulate`, and `Mercury::accumulate`.

Result:

```
Err(BindError::DuplicateTrigger)
```

The child `Typing` cannot rebind `esc` because the ancestor `Mercury` bound it first. Children cannot clobber parents.
