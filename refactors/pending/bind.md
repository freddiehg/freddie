# bind: the #[bind] derive

This doc is the implementation detail of accumulation in the binding layer: the `#[binds]`/`#[bind]` attributes, the `EventHandler<M>` impls the derive generates, and how the active trigger set is accumulated over a laserbeam tree. No clobbering: a trigger is bound at most once on any active path.

bind is a crate within freddie: `bind` (the traits and the walks) plus `bind_macro` (the `#[derive(Bind)]`). It builds on laserbeam: bindability follows laserbeam's descent edges (see "What is bindable").

## The marker and the two traits

The app implements `Bindings` on one marker type, naming the unified `Trigger` enum and the event and output types every handler uses:

```rust
trait Bindings {
    type Trigger: Hash + Eq + Clone;   // enum Trigger { Keyboard(Keyboard), Foreground(Foreground), .. }
    type Event;
    type Output;
}

struct MercuryStruct;
impl Bindings for MercuryStruct {
    type Trigger = Trigger;
    type Event   = MercuryEvent;
    type Output  = Option<Vec<MercuryEffect>>;
}
```

Every bindable node implements `EventHandler<M>`, generated by `#[derive(Bind)]`:

```rust
trait EventHandler<M: Bindings> {
    // add this node's triggers, and those of its active descendants, to `out`.
    fn accumulate(&self, out: &mut HashSet<M::Trigger>);
}
```

## The attributes

```rust
#[derive(Laserbeam, Bind)]
#[binds(MercuryStruct)]
#[bind(Keyboard::new('g'), on_g)]
#[bind(Keyboard::new('y'), on_y)]
struct Inner {
    #[resolve_into] child: Child,   // the one bindable sub-node
    label: String,                  // plain state, not bindable
}
```

- `#[binds(MercuryStruct)]` names the marker `M`. The derive emits `impl EventHandler<MercuryStruct> for Inner`. Because `M` is concrete, `M::Trigger`/`Event`/`Output` are concrete in the impl, so `.into()` lifts each trigger into `Trigger` and the handler signatures `fn(MercuryEvent, Path<Inner, _>) -> Option<Vec<MercuryEffect>>` type-check.
- `#[bind(trigger, handler)]`, repeatable. `trigger` lifts into `M::Trigger` via `Into`; `handler` is `fn(M::Event, Path<Self, Parent>) -> M::Output`.

## What is bindable

Bindability follows laserbeam's descent edges, nothing else:

- For a struct, the bindable children are its `#[resolve_into]` fields. Plain fields (`String`, counters, buffers) are state; they are never recursed into and never implement `EventHandler`.
- For an enum, the bindable child is the active variant's inner type. Every variant's inner type implements `EventHandler<M>` (it derives `Bind`).

`EventHandler<M>` is implemented only on the node types that derive `Bind`, and accumulation follows only `#[resolve_into]` fields and variant inners, which is exactly laserbeam's resolve graph.

## Accumulation

`accumulate` produces the `HashSet<M::Trigger>` to register for the current state, walking the active path's descent edges. The derive generates it per node:

```rust
// struct with its own binds and one #[resolve_into] child:
impl EventHandler<MercuryStruct> for Inner {
    fn accumulate(&self, out: &mut HashSet<Trigger>) {
        for t in [Keyboard::new('g').into(), Keyboard::new('y').into()] {
            if !out.insert(t) { /* no clobbering: a trigger bound twice on the active path is an error */ }
        }
        self.child.accumulate(out);   // recurse the #[resolve_into] edge; self.label is ignored
    }
}

// enum: own (enum-level) binds, then the active variant's inner:
impl EventHandler<MercuryStruct> for Layer {
    fn accumulate(&self, out: &mut HashSet<Trigger>) {
        // ... any enum-level #[bind]s ...
        match self {
            Layer::Nav(n)    => n.accumulate(out),
            Layer::Typing(t) => t.accumulate(out),
        }
    }
}
```

- No clobbering: `HashSet::insert` returning `false` means the trigger is already bound elsewhere on the active path, which is an error, raised here.

Entry point on the root: `root.accumulate::<MercuryStruct>()` allocates the set, calls the trait method, and returns the `HashSet<Trigger>`.
